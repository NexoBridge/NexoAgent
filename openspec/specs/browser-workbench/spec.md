## Requirements
### Requirement: 对话旁浏览器视图
系统 SHALL 提供一个可在对话界面旁显示共享浏览器会话的视图，使用户能在同一屏幕中看到网页和当前会话，而不是进入独立的浏览器功能工作台。

#### Scenario: 打开对话旁浏览器视图
- **当** 用户打开浏览器视图时
- **则** 系统应当显示共享 Electron 浏览器页面
- **并且** 同一视图中应当复用现有对话组件
- **并且** 不应当展示“隐藏工作台”按钮、独立 Agent 提示面板、手动 action 调试控件或功能说明卡片

#### Scenario: 浏览器视图复用当前会话
- **当** 用户在浏览器视图右侧发送消息时
- **则** 系统应当使用现有 Agent runtime 和当前会话历史
- **并且** Agent 应当能够通过 `browser_action` 操作左侧共享浏览器页面

### Requirement: 最小浏览器导航条
对话旁浏览器视图 SHALL 只提供必要的人工导航控件，避免把隐藏浏览器能力包装成独立功能面板。

#### Scenario: 地址栏显示当前 URL
- **当** 浏览器视图加载或页面导航完成时
- **则** 地址栏应当显示当前页面 URL

#### Scenario: 用户从地址栏导航
- **当** 用户在地址栏输入 HTTP 或 HTTPS URL 并提交时
- **则** 系统应当在共享浏览器会话中加载该 URL
- **并且** 地址栏应当在导航完成后同步为实际 URL

#### Scenario: 刷新当前页面
- **当** 用户点击刷新控件时
- **则** 系统应当刷新当前浏览器页面

#### Scenario: 前进和后退
- **当** 用户点击后退或前进控件时
- **则** 系统应当在当前浏览器历史中后退或前进
- **并且** 当对应历史不可用时控件应当禁用或显示不可用状态

### Requirement: 浏览器与对话分栏布局
对话旁浏览器视图 SHALL 分为浏览历史、浏览器内容区域、竖排缩放控制条、会话历史和会话区域。

#### Scenario: 左侧显示历史记录
- **当** 用户在浏览器视图中访问过一个或多个 Web 页面后
- **则** 浏览器区域左侧应当显示最近访问页面的历史记录
- **并且** 用户点击历史记录项时应当在共享浏览器会话中重新打开该页面

#### Scenario: 左侧显示浏览器内容
- **当** 浏览器视图处于打开状态时
- **则** 浏览器主区域应当显示共享 Electron 浏览器会话中的真实 Web 页面内容

#### Scenario: 右侧显示现有会话组件
- **当** 浏览器视图处于打开状态时
- **则** 右侧区域应当显示会话历史栏和与主聊天一致的会话组件
- **并且** 不应当提供独立于主聊天的临时 Agent prompt 表单

#### Scenario: 竖排按钮缩放网页
- **当** 用户点击浏览器区域和会话区域之间的竖排控制按钮时
- **则** 系统应当支持网页放大、缩小和重置缩放
- **并且** 该缩放语义应当类似 `Ctrl + 鼠标滚轮`
- **并且** 控制按钮应当作为占位放在浏览器和会话之间

#### Scenario: 用户拖拽调整分栏大小
- **当** 用户拖拽浏览器区域和会话区域之间的分隔条时
- **则** 系统应当调整两侧宽度
- **并且** 调整过程中不应当导致地址栏、网页内容或会话输入区域互相重叠

#### Scenario: 记住分栏大小
- **当** 用户调整过分栏大小并重新打开浏览器视图时
- **则** 系统应当恢复最近保存的分栏宽度偏好

### Requirement: 对话控制共享浏览器会话
系统 SHALL 让当前对话通过共享浏览器会话控制页面，并将页面视觉结果作为对话内容的一部分返回。

#### Scenario: AI 操作影响浏览器页面
- **当** 用户在会话中请求点击、输入、滚动或导航当前页面时
- **则** Agent 应当通过 `browser_action` 操作共享浏览器会话
- **并且** 左侧浏览器内容应当反映操作后的页面状态

#### Scenario: 手动页面状态可供 AI 读取
- **当** 用户在浏览器区域手动导航或登录后
- **则** 后续 `browser_action` 的 `snapshot` 应当读取同一页面状态

#### Scenario: 截图进入对话
- **当** Agent 在当前对话中调用 `browser_action` 的 `screenshot` action 时
- **则** 系统应当把截图 artifact 附加到本轮 assistant 消息
- **并且** 对话历史重新加载后该截图附件仍应当可见

#### Scenario: 离开浏览器视图保留会话
- **当** 用户切换到其他 Nexo Agent 视图后再返回浏览器视图时
- **则** 浏览器视图应当保留共享会话的当前页面或恢复最近状态

### Requirement: 浏览器视图安全隔离
浏览器视图 SHALL 将外部 Web 内容与 Nexo Agent 应用 UI 和本地权限隔离。

#### Scenario: 外部页面没有 Node 权限
- **当** 浏览器区域加载外部 Web 页面时
- **则** 外部页面不应当获得 Node integration 或 Nexo preload API 权限

#### Scenario: 拦截不受控的新窗口
- **当** 外部页面尝试打开新窗口或导航到非 Web 协议时
- **则** 系统应当按安全策略拦截、拒绝或交给系统浏览器处理
