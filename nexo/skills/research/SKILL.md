# Research

## Process

### 1. Decompose
Break the question into 3–6 specific subclaims or sub-questions that, if answered, fully resolve the main question.

### 2. Search in Parallel
Run multiple WebSearch queries simultaneously — one per subclaim or angle. Vary phrasing across queries to avoid echo-chamber results.

For each source:
- Note the domain and publication date.
- Flag if the source is a primary source (docs, paper, official announcement) vs. secondary (blog, summary).

### 3. Synthesize
- Cross-check key facts across at least 2 independent sources before stating them as confirmed.
- Where sources conflict, report both positions and explain the discrepancy if possible.
- Prefer primary sources over secondhand summaries.

### 4. Uncertainty
Be explicit about confidence level:
- **Confirmed** — multiple independent sources agree.
- **Likely** — single credible source or strong inference.
- **Uncertain** — conflicting evidence or insufficient sources.
- **Unknown** — no reliable information found.

Never present uncertain claims as facts. When knowledge cutoff matters, say so.

### 5. Output
- Lead with the direct answer to the user's question.
- Follow with supporting evidence, sources listed at the end.
- Keep it concise — one clear answer > a wall of hedges.
